# LoRA 微调技术笔记

## 背景

LoRA 是参数高效微调方法的一种，全称是 Low-Rank Adaptation。它的出发点是：大模型在下游任务微调时，并不一定需要更新全部参数，可以在注意力层或前馈层旁路注入低秩矩阵，只训练少量新增参数，从而显著降低显存占用和训练成本。

## 方法原理

在一个线性层中，原始权重矩阵保持冻结，LoRA 额外学习两个低秩矩阵 A 和 B，用它们的乘积近似权重更新量。由于秩 r 通常远小于原始矩阵维度，训练参数量会大幅下降。推理时可以将 LoRA 权重合并回原始权重，也可以按适配器形式动态加载。

## 工程价值

LoRA 的优势主要体现在三个方面。第一，训练显存降低，因为优化器状态和梯度只围绕少量参数维护。第二，多任务适配更灵活，不同任务可以保存不同 LoRA 适配器。第三，迭代成本低，适合快速验证 Prompt、数据清洗和任务定义是否合理。

## 与知识蒸馏的关系

LoRA 和知识蒸馏解决的问题不同。LoRA 更关注如何低成本地让模型适配任务；知识蒸馏更关注如何把教师模型能力迁移给更小的学生模型。两者可以结合使用，例如用教师模型生成软标签，再用 LoRA 微调学生模型，让训练成本和推理成本同时下降。

## 注意事项

LoRA 的效果与注入层位置、秩 r、学习率、数据质量和训练轮数有关。秩过低可能限制表达能力，秩过高会增加显存和过拟合风险。对于中文分类任务，通常需要关注类别描述是否清楚、样本是否均衡，以及验证集指标是否稳定。
