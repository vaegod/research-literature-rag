# Security Policy

## 支持范围

当前项目是本地优先的科研 RAG 示例，主要面向学习、作品集展示和小规模 Demo。安全修复优先覆盖 `main` 分支。

## 敏感信息

请不要提交以下内容：

- `.env` 或任何真实 API Key。
- 本地 `.venv/`、日志、缓存和评测输出。
- 未确认可再分发许可的真实论文 PDF。
- 来自不可信来源的 FAISS 索引文件。

如果 API Key 曾经出现在公开仓库、截图、日志或压缩包中，请立即到服务商后台轮换。

## FAISS 索引加载

LangChain 的 FAISS 本地索引会通过 pickle 加载文档元数据，因此本项目通过 `TRUST_LOCAL_FAISS_INDEX` 显式声明信任边界。只应加载由本项目在本机生成的索引，不要加载陌生人提供的 `index.pkl`。

## 报告问题

如果发现安全问题，请不要在公开 issue 中贴出密钥、日志或攻击细节。可以先创建一个不含敏感信息的 issue，说明受影响模块和复现条件。
