# 实现边界说明

本项目提供一个本地优先的科研文献 RAG 应用样例，覆盖文档管理、知识库构建、混合检索、引用来源、Agent 工作流、评测接口和 Web 控制台。以下内容用于说明实现范围和使用边界。

## 1. 已实现能力

| 模块 | 实现范围 |
| --- | --- |
| 文档管理 | 支持 PDF、Markdown、TXT 上传；支持 Markdown/TXT 创建、编辑、删除；PDF 只读 |
| 知识库构建 | 支持文本抽取、chunk 切分、元数据注入、Embedding 向量化、FAISS 本地索引 |
| 检索策略 | 支持 FAISS dense 检索、BM25 关键词检索、RRF/hybrid 融合、section rerank、可选 SiliconFlow reranker |
| RAG 问答 | 支持普通 RAG 问答和 SSE 流式输出，返回 `sources` 与 `related_chunks` |
| Agent 工作流 | 使用 LangGraph 编排文献问答、论文对比、实验查询、阅读笔记生成和普通问答 |
| 评测 | 支持 source hit、keyword hit、Recall@K、MRR、平均耗时和失败样例输出 |
| 前端 | 支持文档上传、索引重建、问答模式切换、引用展示、chunk 浏览和检索调试信息展示 |
| 工程化 | 提供 Pytest、Ruff、Docker Compose、GitHub Actions、Dependabot 和敏感文件检查脚本 |

## 2. 依赖条件

- 端到端 RAG 和 Agent 问答依赖 OpenAI-compatible Chat/Embedding 服务。
- 默认配置使用 SiliconFlow 兼容接口，也可以替换为其他兼容服务。
- `ENABLE_RERANKER=true` 时会额外调用 rerank 接口；默认关闭，便于本地测试。
- 构建 FAISS 索引前需要先准备 `data/raw_docs/` 中的源文档。

## 3. 索引同步

文档增删改后，后端接口会返回 `index_status=stale`。Web 控制台会在文档变更后调用 `/knowledge/build` 同步索引；如果通过 Swagger、脚本或第三方客户端直接调用 CRUD 接口，需要自行触发重建。

这种设计将“修改源文档”和“重建索引”分成两个显式步骤，便于在批量导入文档时控制重建时机。

## 4. Agent 路由

LangGraph 负责工作流编排。默认路由器使用规则匹配，保证本地测试可复现；设置 `AGENT_ROUTER_MODE=hybrid` 或 `llm` 后，可启用模型路由作为兜底。

该实现更接近“工作流型 Agent”：路径可解释、节点边界清晰，适合业务应用原型和小规模验证环境。它不是完全自主规划型 Agent。

## 5. 引用可信度

接口返回的 `sources` 和 `related_chunks` 来自检索结果，前端会把它们与回答一起展示。引用来源用于核查答案依据，但系统没有逐句验证模型生成内容与每个引用片段之间的蕴含关系。严格场景下可以增加 citation grounding 或人工复核。

## 6. PDF 与数据边界

- PDF 解析基于 `pypdf.extract_text()`，适合文本型 PDF。
- 扫描版 PDF 需要额外 OCR，不在默认链路中。
- 真实论文 PDF 可能存在版权或再分发限制，默认不提交到 GitHub。
- `data/raw_docs/*.md` 是样例笔记，主要用于复现 RAG 流程。

## 7. 测试边界

无 API Key 场景下，测试使用 mock 覆盖基础路由、schema、loader、检索结果格式、Agent 路由和评测数据一致性。真实 Chat/Embedding/Reranker 服务、Docker 启动和完整端到端生成效果需要在配置密钥后额外验证。

## 8. 可扩展方向

- 引入更细粒度的引用校验或 answer grounding。
- 为上传、索引构建和模型调用增加更完整的错误分类。
- 增加 deterministic fake embedding 的集成测试，覆盖无外部 API 的真实建库和检索。
- 增加会话记忆、用户级知识库隔离和权限边界。
- 支持 OCR、论文表格/图片解析和跨文档证据图谱。
