# 仓库维护说明

本说明用于保持仓库内容干净、可复现、可审阅。项目默认只提交源码、样例 Markdown、配置模板、测试和文档；密钥、本地环境、索引文件和真实论文 PDF 不进入版本控制。

## 1. 敏感信息

不要提交以下内容：

- `.env` 或任何真实 API Key。
- 本地 `.venv/`、日志、缓存和评测输出。
- 未确认可再分发许可的真实论文 PDF。
- 来自不可信来源的 FAISS 索引文件。

`.env.example` 只保留占位值，例如 `your_api_key`。如果 API Key 曾经出现在公开仓库、截图、日志或压缩包中，需要到服务商后台轮换。

推荐检查命令：

```powershell
rg "sk-|api[_-]?key|secret|token" -g "!.env" -g "!.venv/**" -g "!__pycache__/**"
python scripts/open_source_audit.py
```

## 2. 论文与数据

- `data/raw_docs/*.md` 用作 RAG 流程样例，可以提交。
- `data/raw_docs/*.pdf` 默认忽略，不直接提交真实论文 PDF。
- 如需使用真实论文，可按 [真实论文 Demo](真实论文Demo.md) 的方式放在本地后重建索引。
- 实验记录如果包含真实项目、姓名、单位或未公开指标，需要脱敏后再提交。

## 3. 生成物与缓存

以下内容属于本地运行产物，不进入仓库：

- `.venv/`
- `.pytest_cache/`
- `__pycache__/`
- `vector_store/faiss_index/*`
- `data/processed_docs/*`
- `data/eval/eval_result.csv`

## 4. 本地验证

```powershell
.\.venv\Scripts\python.exe -m pytest
ruff check .
python scripts/open_source_audit.py
docker compose config --quiet
```

有 API Key 时，可以额外验证端到端路径：

```powershell
python scripts/build_index.py
python scripts/run_eval.py --limit 5
uvicorn app.main:app --host 0.0.0.0 --port 8010 --reload
```

## 5. 文档一致性

文档更新时需要关注以下内容：

- README 中的启动命令、API 入口、Swagger 地址和前端地址是否仍然准确。
- RAG、Agent、索引、chunk、评测指标的描述是否与代码行为一致。
- 样例 Markdown 与真实 PDF 的区别是否说明清楚。
- 已实现能力和实现边界是否分开表达。
- 截图是否来自当前界面，避免展示过期交互。

## 6. GitHub 配置

- `LICENSE` 与 README 中的许可证描述保持一致。
- `CONTRIBUTING.md`、`SECURITY.md`、`CODE_OF_CONDUCT.md` 保持简洁。
- PR 模板覆盖变更说明、验证方式和风险说明。
- GitHub Actions 覆盖 audit、lint、test 和 Docker build。
- Dependabot 用于依赖和 GitHub Actions 更新提醒。
