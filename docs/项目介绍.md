# 项目介绍

“科研文献智能问答与实验分析助手”是一个面向论文阅读和实验记录管理的 RAG 应用。系统将本地 PDF、Markdown、TXT 文档解析为带来源信息的 chunk，写入 FAISS 向量索引，并通过混合召回和章节重排提升检索质量。用户可以在 Web 控制台中上传资料、重建索引、进行 RAG 问答、查看引用来源，也可以通过 Agent 工作流完成论文对比、实验记录查询和阅读笔记生成。

## 一句话概览

基于 FastAPI、LangChain、LangGraph 和 FAISS 的科研资料问答系统，支持文档知识库构建、引用溯源、混合检索、流式 RAG、实验记录查询和轻量评测。

## 核心链路

```text
上传文档
  -> 文档解析
  -> 文本切分
  -> Embedding 向量化
  -> FAISS + BM25 混合检索
  -> section/reranker 精排
  -> LLM 生成回答
  -> 返回引用来源和相关 chunk
```

Agent 工作流在 RAG 基础上增加意图路由：

```text
用户问题
  -> 意图识别
  -> 文献问答 / 论文对比 / 实验查询 / 阅读笔记 / 普通问答
  -> 工具调用
  -> 统一响应
```

## 技术要点

- 文档处理：支持 PDF、Markdown、TXT，保留 `source`、`page`、`chunk_id`、`section` 等元数据。
- 检索策略：结合 FAISS dense 检索和 BM25 关键词检索，并使用 RRF 和归一化分数融合结果。
- 排序优化：根据问题意图对不同论文 section 加权，减少参考文献页和噪声片段进入 TopK。
- 引用溯源：接口返回 `sources` 和 `related_chunks`，前端可直接查看答案依据。
- 工作流编排：LangGraph 将不同问题类型路由到独立工具节点，避免所有任务挤在单一 Prompt 中。
- 评测闭环：评测集记录预期来源、关键词和 gold chunk，支持 source hit、keyword hit、Recall@K、MRR 等指标。
- 工程交付：提供 Swagger、Docker Compose、GitHub Actions、Ruff、Pytest 和敏感文件检查脚本。

## 适用场景

- 阅读论文时快速定位贡献、方法、实验设置和局限。
- 管理个人论文笔记、实验记录和技术调研资料。
- 对比多个主题，例如 LoRA、知识蒸馏、RAG 等概念差异。
- 检查 RAG 回答是否来自可追溯的文档片段。

## 实现边界

- 适合本地单人知识库和小规模资料管理，不包含用户登录、多租户隔离和权限系统。
- PDF 解析依赖文本抽取，扫描版 PDF 需要额外 OCR。
- 评测指标用于开发回归和问题定位，不替代人工评审。
- Web 控制台会在文档变更后触发索引重建；直接调用后端 CRUD 接口时，需要根据 `index_status=stale` 显式调用 `/knowledge/build`。
